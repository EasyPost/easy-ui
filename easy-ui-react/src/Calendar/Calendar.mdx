# Calendar

Calendar displays a grid of days and allows users to select a single date.

<Canvas of={CalendarStories.Default} />

## Default Value

A `<Calendar />` component allows users to set default selected date (uncontrolled).

<Canvas of={CalendarStories.DefaultValue} />

## Set minimum and maximum value

Set minimum and maximum allowed date that a user may select. Dates outside the minimum and maximum value are disabled and unreachable.

<Canvas of={CalendarStories.LimitAvailableDates} />

## Show days outside of current month

Display the days falling into the other months.

<Canvas of={CalendarStories.ShowDaysOutsideCurrentMonth} />

<Controls
  of={CalendarStories.ShowDaysOutsideCurrentMonth}
  include={["showDaysOutsideCurrentMonth"]}
/>

## Set Date Availability

Callback that is called for each date of the calendar. If it returns `true`, then the date is unavailable.

<Canvas of={CalendarStories.DatesAvailability} />

## Disabled Calendar

Use `isDisabled` to disabled the calendar entirely.

<Canvas of={CalendarStories.DisabledCalendar} />

<Controls
  of={CalendarStories.DisabledCalendar}
  include={["isDisabled", "showDaysOutsideCurrentMonth"]}
/>

## Read-Only Calendar

Use `isReadOnly` to make the calendar immutable.

<Canvas of={CalendarStories.ReadonlyCalendar} />

<Controls
  of={CalendarStories.ReadonlyCalendar}
  include={["isReadOnly", "showDaysOutsideCurrentMonth"]}
/>

## Controlled Selection

For controlled selection, use `onChange` and `value`. Uncontrolled selection is possible with `defaultValue`. See associated code snippet.

<Canvas of={CalendarStories.ControlledSelection} />

## Invalid Selection

An error message is displayed when the selected value is invalid according to application logic. Select a weekend date to see an invalid selection.

<Canvas of={CalendarStories.InvalidSelection} />
